# BuyBot v2 — Feature Batch Implementation Plan

> **For agentic workers:** Executed via superpowers:subagent-driven-development. Full task text is provided in each subagent dispatch (not read from this file).

**Goal:** Fix the negotiation price-parsing bug and add 8 features: responsive desktop layout, image-upload search, budget range, real-time location, hover feedback, real login/auth, settings (language + default address), and message auto-translation.

**Locked decisions:**
- **Login:** Real backend auth — FastAPI users, password hashing, token, persistence (SQLite).
- **Translation:** Gemini for live negotiation messages (German↔user language), shown as a lighter line under the original; static UI via an i18n string table.
- **Image:** Gemini Vision identifies an uploaded photo and fills the search query.
- **Budget:** A `[min, max]` range that both filters listings and feeds price scoring (replaces the single `budget` end-to-end). Onboarding language default: English.

**Tech:** existing stack (FastAPI + LangGraph + Gemini; React + Vite + Tailwind v4). Add SQLite (stdlib `sqlite3`) + `passlib[bcrypt]` for auth.

---

## Task order (dependency-aware, sequential)

### Backend
- **B1 — Fix seller price parse bug.** Seller "€165 … für €179" is read as €165. Trust the mock seller's structured price over the classifier's parsed price; improve `_parse_price` to prefer the price following "für"/the last price. Test.
- **B2 — Budget range.** Replace `budget: float` with `budget_min: float` + `budget_max: float` in `state.py`/`initial_state`, `analyst.py` (filter to `[min,max]`, price score across range), `main.py` `SessionRequest`. Update tests + the live driver fields.
- **B3 — Real auth + per-user settings.** SQLite users table; `POST /auth/signup`, `POST /auth/login` → bearer token; `GET/PUT /settings` (language, default_address) keyed by user. bcrypt hashing.
- **B4 — Translation endpoint.** `POST /translate {text, target_lang}` → Gemini → `{translation}`. Small in-memory cache.
- **B5 — Vision endpoint.** `POST /vision/identify {image_base64}` → Gemini Vision → `{query}`.

### Frontend
- **F1 — Responsive desktop layout + hover states.** Make all screens work landscape/desktop (wider max-width, multi-column where it helps); add clear hover/active feedback to buttons & cards.
- **F2 — i18n foundation.** Lightweight i18n provider + EN/DE/ZH string table; language switch; default English.
- **F3 — Login/Signup page + auth gating** + api client auth (token storage, attach header).
- **F4 — Settings page** (language selector + default address; prefill InputScreen location from default address).
- **F5 — Message auto-translation** under seller (and buyer) messages — lighter translated line via `/translate`.
- **F6 — Image upload** in InputScreen → `/vision/identify` → fill query.
- **F7 — Budget range dual-slider + real-time geolocation** in InputScreen (navigator.geolocation with permission; send `budget_min`/`budget_max`).

Each task: implement → spec review → code-quality review → commit. Build/tests green before commit.
